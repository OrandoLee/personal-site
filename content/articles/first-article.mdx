这个网站从一个很小的需求开始：我需要一个地方，把文章、图片、视频和那些还没变成作品的想法放在一起。

它不是简历页，也不只是作品集。它更像一个持续整理的工作台：有完成的内容，也有过程中的线索。

## 为什么是每日档案

作品集通常强调完成度，日记通常强调真实感。我希望这个网站可以同时保留这两种东西。

> 一个有用的个人网站，不只要让最新作品容易被看见，也应该保存那些通往作品的路径。

首页的数据来自统一的更新结构。以后添加一条创作记录，只需要编辑 `data/updates.ts`：

```ts
{
  id: "daily-2026-05-30-image",
  title: "铬色花园海报",
  type: "image",
  date: "2026-05-30",
  description: "一次视觉练习。",
  cover: "/images/chrome-garden.svg",
  link: "/gallery#chrome-garden"
}
```

## 阅读和展示

文章使用本地 Markdown / MDX 管理。详情页支持标题、图片、代码块、引用、列表和表格。

![档案切片](/images/archive-fragment.svg)

### 页面分工

| 板块 | 用途 |
| --- | --- |
| 首页 | 每日更新与最新内容 |
| 文章 | 文本作品和长期想法 |
| 画廊 | 图片、视频与视觉实验 |
| Orask | 访客反馈入口 |

这个结构的重点是可持续：添加文件，更新数据，然后继续创作。
